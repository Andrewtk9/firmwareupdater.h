# Firmwareupdater

A lightweight ESP32 library for **Over-The-Air (OTA) firmware updates** and **remote debug logging** using a custom HTTP WebService.

## ‚úÖ What It Does

This library performs:

1. **Firmware update checks** by pinging a remote server.
2. **Remote debug logging** by sending messages as JSON to a debug endpoint.
3. **Dynamic debug activation**: logging only occurs if the server responds with `"debug": true`.

---

## üöÄ Key Features

- OTA firmware update via HTTP
- Remote debug logging with metadata (file, line, function)
- Device identifies itself using ESP32's chip ID
- Automatically controlled debug mode via ping response
- Works with any custom backend
- Simple API and macro for easy logging

---

## üõ†Ô∏è How It Works

### üîÅ Periodic Ping

The ESP32 periodically makes a request to:
GET
[https://yourserver.com/atualizador/ping/]()`<chipId>`/`<version> `**Server Response Example:**

```json
{
  "debug": true,
  "atualizar": "update123",    // or null if no update
  "tempoPing": 300000,         // time in milliseconds until next ping
  "status": "ok"
}
```

* If `atualizar` has a value, the library downloads and installs the new firmware.
* If `debug` is `true`, future log messages will be sent to the server.

---

## üß© Server Endpoints

| Method | Endpoint                                         | Description                         |
| ------ | ------------------------------------------------ | ----------------------------------- |
| GET    | `/atualizador/ping/<chipId>/<version>`         | Checks for update and debug status  |
| GET    | `/atualizador/atualizar/<updateId>`            | Returns the binary firmware         |
| GET    | `/atualizador/confirmarAtualizacao/<updateId>` | Confirms update success             |
| POST   | `/debug-log`                                   | Receives debug log messages in JSON |
